# 本番デプロイ — 要求内容

## 概要

Phase 1〜3 の実装が完了し、MVP のコードが揃った。
本番環境を構築してサービスを公開し、仮説検証（ユーザー登録・継続率）を開始する。

| -------- | ---------------- | ------ |
| Web      | Vercel           | Hobby  |
| API      | Vercel           | Hobby  |
| DB       | Supabase         | Free   |

## 各サービスの要件

### Supabase（DB）

- PostgreSQL データベースを作成する
- Prisma マイグレーションを本番DBに適用する（`prisma migrate deploy`）
- 接続文字列（`DATABASE_URL`）を取得して API の環境変数に設定する

### Vercel（API）

- Hono API をデプロイする
- **先に Web（フロントエンド）をデプロイして Vercel の URL を確定させてから、API の環境変数に設定する**
- 以下の環境変数を設定する：
  - `DATABASE_URL`（Supabase 接続文字列）
  - `JWT_SECRET`（強力なランダム文字列）
  - `JWT_EXPIRES_IN=15m`
  - `REFRESH_TOKEN_EXPIRES_IN=7d`
  - `CORS_ORIGIN`（Web の Vercel デプロイ URL）
  - `NODE_ENV=production`
- デプロイ後に `/health` エンドポイントの疎通確認をする

### Vercel（Web）

- Next.js アプリをデプロイする
- 以下の環境変数を設定する：
  - `NEXT_PUBLIC_API_URL`（Render のデプロイ URL）
- デプロイ後にアプリにアクセスできることを確認する

---

## 動作確認項目

- [ ] ユーザー登録 → ログイン → ログアウトが動作する
- [ ] 日記の作成 → 一覧表示 → 編集 → 削除が動作する
- [ ] テンプレートの作成 → デフォルト設定 → 日記作成時に自動適用される
- [ ] ページリロード後もログイン状態が維持される

---

## 制約事項

- カスタムドメインは対象外（MVP 後に対応）
- Vercel Hobby プランのサーバーレス関数実行時間は 10 秒まで（現状の API 処理は問題なし、AI 分析機能追加時は要再検討）
- スリープ対策は行わない（Vercel サーバーレスはスリープなし）
- Supabase Free プランの上限（500MB ストレージ、50,000 MAU）を把握しておく
