# Phase 3: UI/UX 改善 + デプロイ準備 — 要求内容

## 概要

Phase 1（認証）と Phase 2（日記 CRUD）が完了し、MVP の機能は揃った。
PRD のペルソナの利用シーンが「寝る前にスマホで記録」であるため、レスポンシブ対応なしではリリースできない。
Phase 3 でテンプレート機能 + ヘッダー/フッター + レスポンシブ + デプロイ準備を行い、早期リリースして仮説検証を開始する。

---

## ユーザーストーリー

| #   | ストーリー                                                                   |
| --- | ---------------------------------------------------------------------------- |
| 1   | ユーザーとして、日記のテンプレートを作成・管理して毎回の入力を効率化したい   |
| 2   | ユーザーとして、デフォルトテンプレートを設定して日記作成時に自動適用させたい |
| 3   | ユーザーとして、ヘッダーのナビゲーションから各機能にすぐアクセスしたい       |
| 4   | ユーザーとして、スマホでも快適に日記を書きたい（レスポンシブ対応）           |
| 5   | ユーザーとして、本番環境でサービスを利用したい（デプロイ）                   |

---

## テンプレート API 機能要件

認証済みユーザーのみアクセス可能。自分のテンプレートのみ操作できる。

### エンドポイント一覧

| メソッド | パス                         | 説明                       | ステータス |
| -------- | ---------------------------- | -------------------------- | ---------- |
| GET      | `/api/templates`             | テンプレート一覧取得       | 200        |
| POST     | `/api/templates`             | テンプレート作成           | 201        |
| GET      | `/api/templates/default`     | デフォルトテンプレート取得 | 200        |
| GET      | `/api/templates/:id`         | テンプレート詳細取得       | 200        |
| PUT      | `/api/templates/:id`         | テンプレート更新           | 200        |
| DELETE   | `/api/templates/:id`         | テンプレート削除           | 200        |
| PUT      | `/api/templates/:id/default` | デフォルトに設定           | 200        |

### テンプレート一覧 `GET /api/templates`

- ユーザーの全テンプレートを返す
- createdAt 降順でソート

### テンプレート作成 `POST /api/templates`

- 必須: name（最大50文字）, content（最大10000文字）
- バリデーション: `packages/shared` の `createTemplateSchema` を使用

### デフォルトテンプレート取得 `GET /api/templates/default`

- `isDefault: true` のテンプレートを返す
- デフォルト未設定の場合は 404

### テンプレート更新 `PUT /api/templates/:id`

- 部分更新: name, content（いずれも optional）
- バリデーション: `packages/shared` の `updateTemplateSchema` を使用

### テンプレート削除 `DELETE /api/templates/:id`

- 物理削除
- デフォルトテンプレートも削除可能（デフォルトが解除される）

### デフォルト設定 `PUT /api/templates/:id/default`

- 指定テンプレートを `isDefault: true` に設定
- 他のテンプレートの `isDefault` を `false` に更新（1ユーザー1デフォルト）

## Web テンプレート機能要件

### ページ一覧

| パス                   | 説明             | 種別      |
| ---------------------- | ---------------- | --------- |
| `/templates`           | テンプレート一覧 | protected |
| `/templates/new`       | テンプレート作成 | protected |
| `/templates/[id]/edit` | テンプレート編集 | protected |

### コンポーネント

| コンポーネント | 説明                                             |
| -------------- | ------------------------------------------------ |
| `TemplateForm` | 作成・編集共通フォーム（EntryForm と同パターン） |
| `TemplateCard` | 一覧用カード（名前 + デフォルトマーク）          |

### テンプレート自動適用

- 日記新規作成ページ（`/entries/new`）読み込み時に `getDefaultTemplate()` を呼び出し
- 取得したテンプレートの `content` を EntryForm の初期値として設定

---

## ヘッダー/フッター機能要件

### Header

- アプリ名（ダッシュボードへのリンク）
- ナビゲーション: ダッシュボード / 日記一覧 / カレンダー / テンプレート
- ユーザー情報 + ログアウトボタン
- モバイル: ハンバーガーメニューでナビ開閉

### Footer

- コピーライト表示（最小限）

### Protected レイアウト統合

- 既存のインラインヘッダーを `<Header />` に置換
- `<Footer />` を追加

## デプロイ要件

### リリース構成

| サービス | プラットフォーム | プラン |
| -------- | ---------------- | ------ |
| Web      | Vercel           | 無料枠 |
| API      | Render           | 無料枠 |
| DB       | Supabase         | 無料枠 |

### 対応事項

- `.env.example` に `CORS_ORIGIN` を追加
- CORS 設定の本番対応確認
- Cookie 設定の本番対応確認（`SameSite=None; Secure` の可能性）

---

## セキュリティ要件

- テンプレート API: 認証ミドルウェア適用、自分のテンプレートのみ操作可能
- 他ユーザーのテンプレートは 404 を返す（存在を漏らさない）
- 入力はサーバーサイドでも Zod バリデーション
- 本番環境の CORS 設定は許可オリジンを限定

---

## 制約事項

- テンプレートのフォルダ・タグ分類は対象外（将来対応）
- テンプレート変数（`{{date}}` 等の自動置換）は対象外（将来対応）
- AI 分析機能は対象外（Phase 4 以降）
- カスタムドメインは対象外（MVP 後に対応）
