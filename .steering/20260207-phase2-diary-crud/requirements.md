# Phase 2: 日記 CRUD — 要求内容

## 概要

Phase 1 で構築した認証基盤の上に、アプリケーションのコア機能である日記の作成・閲覧・編集・削除（CRUD）を実装する。
API（Hono）とフロントエンド（Next.js）の両方を対象とする。

---

## ユーザーストーリー

| #   | ストーリー                                                         |
| --- | ------------------------------------------------------------------ |
| 1   | ユーザーとして、今日の日記を書きたい（日付・評価・一言メモ・本文） |
| 2   | ユーザーとして、過去の日記を一覧で確認したい                       |
| 3   | ユーザーとして、特定の日記の詳細を読みたい                         |
| 4   | ユーザーとして、書いた日記を編集したい                             |
| 5   | ユーザーとして、不要な日記を削除したい                             |
| 6   | ユーザーとして、カレンダー形式で日記の記録状況を把握したい         |

---

## API 機能要件

認証済みユーザーのみアクセス可能。自分の日記のみ操作できる。

### エンドポイント一覧

| メソッド | パス                    | 説明                                       | ステータス |
| -------- | ----------------------- | ------------------------------------------ | ---------- |
| GET      | `/api/entries`          | 日記一覧取得（ページネーション・フィルタ） | 200        |
| POST     | `/api/entries`          | 日記作成                                   | 201        |
| GET      | `/api/entries/calendar` | カレンダー用データ取得（年月指定）         | 200        |
| GET      | `/api/entries/:id`      | 日記詳細取得                               | 200        |
| PUT      | `/api/entries/:id`      | 日記更新                                   | 200        |
| DELETE   | `/api/entries/:id`      | 日記削除                                   | 200        |

### 日記一覧 `GET /api/entries`

- クエリパラメータ: `page`, `limit`, `from`, `to`, `rating`
- デフォルト: page=1, limit=20
- entryDate 降順でソート
- レスポンスに `pagination` オブジェクトを含む（page, limit, total, totalPages）

### 日記作成 `POST /api/entries`

- 必須: entryDate（YYYY-MM-DD形式）
- 任意: content（最大10000文字）, shortMemo（最大200文字）, rating（1-5）
- content, shortMemo, rating のいずれか1つ以上が必要
- バリデーション: `packages/shared` の `createEntrySchema` を使用

### カレンダー `GET /api/entries/calendar`

- クエリパラメータ: `year`, `month`（必須）
- 指定月の日記データを日付ごとに集計（count, avgRating）
- 日記がない日はレスポンスに含めない
- avgRating は小数点第1位で四捨五入

### 日記詳細 `GET /api/entries/:id`

- 自分の日記のみ取得可能
- 他ユーザーの日記・存在しない日記は 404

### 日記更新 `PUT /api/entries/:id`

- 自分の日記のみ更新可能
- 部分更新: 送信されたフィールドのみ更新
- バリデーション: `packages/shared` の `updateEntrySchema` を使用

### 日記削除 `DELETE /api/entries/:id`

- 自分の日記のみ削除可能
- 物理削除（ソフトデリートは将来対応）

## Web 機能要件

### ページ一覧

| パス                 | 説明                             | 種別      |
| -------------------- | -------------------------------- | --------- |
| `/entries`           | 日記一覧（ページネーション付き） | protected |
| `/entries/new`       | 日記作成フォーム                 | protected |
| `/entries/[id]`      | 日記詳細                         | protected |
| `/entries/[id]/edit` | 日記編集フォーム                 | protected |
| `/entries/calendar`  | カレンダービュー                 | protected |

### コンポーネント

| コンポーネント | 説明                                                   |
| -------------- | ------------------------------------------------------ |
| `EntryForm`    | 作成・編集共通フォーム（mode: create/edit）            |
| `EntryCard`    | 一覧用カードコンポーネント（日付・評価・一言メモ抜粋） |
| `EntryDetail`  | 詳細表示コンポーネント                                 |
| `RatingStars`  | 評価表示・入力コンポーネント（1-5の星）                |
| `Calendar`     | カレンダーコンポーネント（月表示・月移動）             |
| `Pagination`   | ページネーションコンポーネント                         |

### ダッシュボード拡張

- 「日記を書く」「カレンダーを見る」のクイックアクションカード
- 最近の日記3件を表示
- 「すべて見る」リンクで一覧ページへ遷移

### フォーム仕様

- 日付: date input（デフォルトは今日の日付）
- 評価: 1-5の星ボタン選択（クリックで設定、再クリックでクリア）
- 一言メモ: テキスト入力（200文字制限、残り文字数表示）
- 本文: textarea（10000文字制限）
- バリデーション: Zod（shared スキーマ）によるクライアントサイドバリデーション
- 送信後: 作成→一覧、編集→詳細にリダイレクト

### 削除確認

- 削除ボタン押下時に確認ダイアログ（モーダル or window.confirm）
- 削除後は一覧にリダイレクト

---

## セキュリティ要件

- すべてのエンドポイントで認証ミドルウェアを適用
- ユーザーは自分の日記のみ操作可能（他ユーザーの日記は 404 を返す）
- 入力はサーバーサイドでも Zod バリデーションを実施
- XSS 対策: React のデフォルトエスケープに依存（dangerouslySetInnerHTML 不使用）

---

## 制約事項

- 本文のマークダウン対応は対象外（AI 分析との相性を考慮し将来対応）
- 画像添付は対象外（将来対応）
- AI 分析機能は対象外（Phase 3 以降）
- ソフトデリートは対象外（将来対応）
- 国際化対応は対象外（MVP 後に対応）
