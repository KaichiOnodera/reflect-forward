# Phase 1 認証機能 - 要求定義

## 今回の作業概要

Phase 1として、Reflect ForwardにJWT認証機能を実装する。

## ユーザーストーリー

| ID     | ストーリー                                                   | 優先度 |
| ------ | ------------------------------------------------------------ | ------ |
| AUTH-1 | 新規ユーザーとして、メールアドレスとパスワードでアカウントを作成したい | 必須   |
| AUTH-2 | 登録済みユーザーとして、ログインしたい                       | 必須   |
| AUTH-3 | ログイン中のユーザーとして、ログアウトしたい                 | 必須   |
| AUTH-4 | ログイン中のユーザーとして、セッションを維持したい           | 必須   |

## 機能要件

| 機能             | 仕様                                               |
| ---------------- | -------------------------------------------------- |
| ユーザー登録     | email（一意）+ password（8文字以上）、任意でdisplayName |
| ログイン         | email + password で認証                            |
| アクセストークン | JWT、15分有効、メモリ保持                          |
| リフレッシュトークン | UUID、7日有効、HttpOnly Cookie + DB保存          |
| パスワード保護   | bcrypt（コスト10）でハッシュ化                     |
| ログアウト       | リフレッシュトークンをDBから削除、Cookie削除       |

## 受け入れ条件

### AUTH-1: ユーザー登録

- [ ] 有効なメールアドレスとパスワードで登録できる
- [ ] 登録後、自動的にログイン状態になる
- [ ] 既存のメールアドレスでは登録できない
- [ ] パスワードが8文字未満の場合、エラーになる

### AUTH-2: ログイン

- [ ] 正しい認証情報でログインできる
- [ ] ログイン後、ダッシュボードにリダイレクトされる
- [ ] 誤った認証情報ではログインできない

### AUTH-3: ログアウト

- [ ] ログアウトボタンでセッションを終了できる
- [ ] ログアウト後、ログインページにリダイレクトされる
- [ ] ログアウト後、保護されたページにアクセスできない

### AUTH-4: セッション維持

- [ ] ブラウザを閉じて再度開いてもログイン状態が維持される（7日間）
- [ ] アクセストークン期限切れ時、自動的にリフレッシュされる

## 制約事項

- 永続ドキュメント（`docs/`）への変更は最小限（`repository-structure.md`のみ）
- 既存のPrismaスキーマ、Zodバリデーション、型定義を活用
- 依存関係（jsonwebtoken, bcryptjs）は既に追加済み
