# Reflect Forward - プロダクト要求定義書 (PRD)

## 1. プロダクトビジョン

### 1.1 背景

適応障害を経験し、自分を見失いかけたことがある。日記を書き、AIと対話しながら内面と向き合う中で、本当にやりたいことを見つけることができた。

同じように悩む人が、自分自身と向き合い、前に進む力を取り戻せるサービスを作りたい。

### 1.2 ビジョンステートメント

> **自分を見失いかけている人が、日々の記録と振り返りを通じて内面と向き合い、本当にやりたいことを見つけられるようになる。**

### 1.3 サービスコンセプト

**Reflect Forward** は「振り返って前進する」をコンセプトにした日記サービス。

- **シンプル**: 一言メモと5段階評価だけでも記録できる
- **継続**: 低いハードルで習慣化を支援
- **発見**: 蓄積されたデータとAI分析で自己理解を深める

### 1.4 プロダクトの価値

| ユーザーの課題                 | 提供する価値                                      |
| ------------------------------ | ------------------------------------------------- |
| 自分が何をしたいのかわからない | 日々の記録から自分のパターンや価値観を可視化      |
| 内面と向き合う方法がわからない | シンプルな記録フォーマットで振り返りを習慣化      |
| 日記を続けられない             | 最小限の入力で記録可能（一言メモ + 評価のみでOK） |
| 自己理解の客観的な手段がない   | AIによる傾向分析と気づきの提供（将来）            |

## 3. 成功指標（KPI）

### 3.1 MVP（Phase 1-2）の検証目標

| 指標             | 目標値   | 測定方法                           |
| ---------------- | -------- | ---------------------------------- |
| ユーザー登録数   | 10名以上 | DBのusersテーブル集計              |
| 7日間継続率      | 50%以上  | 7日間で3回以上記録したユーザー比率 |
| 平均入力完了時間 | 3分以内  | 手動計測（初期はサンプリング）     |
| 致命的バグ       | 0件      | 認証・データ保存に関する障害       |

### 3.2 検証したい仮説

1. **習慣化仮説**: 一言メモ + 評価だけなら、忙しい人でも続けられる
2. **価値仮説**: カレンダービューで過去を振り返ることに価値を感じる
3. **技術仮説**: JWT認証の自前実装でセキュアなサービスを提供できる

### 4.2 Phase 1: 認証機能

#### ユーザーストーリー

| ID     | ストーリー                                                                                             | 優先度 |
| ------ | ------------------------------------------------------------------------------------------------------ | ------ |
| AUTH-1 | 新規ユーザーとして、メールアドレスとパスワードでアカウントを作成したい。自分専用の日記を管理するため。 | 必須   |
| AUTH-2 | 登録済みユーザーとして、ログインしたい。自分の日記にアクセスするため。                                 | 必須   |
| AUTH-3 | ログイン中のユーザーとして、ログアウトしたい。共有PCでの利用を安全に終了するため。                     | 必須   |
| AUTH-4 | ログイン中のユーザーとして、セッションを維持したい。毎回ログインする手間を省くため。                   | 必須   |

#### 機能仕様

| 機能                 | 仕様                                                    |
| -------------------- | ------------------------------------------------------- |
| ユーザー登録         | email（一意）+ password（8文字以上）、任意でdisplayName |
| ログイン             | email + password で認証                                 |
| アクセストークン     | JWT、15分有効、メモリ保持                               |
| リフレッシュトークン | UUID、7日有効、HttpOnly Cookie + DB保存                 |
| パスワード保護       | bcrypt（コスト10）でハッシュ化                          |
| ログアウト           | リフレッシュトークンをDBから削除、Cookie削除            |

#### 受け入れ条件

**AUTH-1: ユーザー登録**

- [ ] 有効なメールアドレスとパスワードで登録できる
- [ ] 登録後、自動的にログイン状態になる
- [ ] 既存のメールアドレスでは登録できない（エラーメッセージ表示）
- [ ] パスワードが8文字未満の場合、エラーになる

**AUTH-2: ログイン**

- [ ] 正しい認証情報でログインできる
- [ ] ログイン後、ダッシュボードにリダイレクトされる
- [ ] 誤った認証情報ではログインできない（エラーメッセージ表示）

**AUTH-3: ログアウト**

- [ ] ログアウトボタンでセッションを終了できる
- [ ] ログアウト後、ログインページにリダイレクトされる
- [ ] ログアウト後、保護されたページにアクセスできない

**AUTH-4: セッション維持**

- [ ] ブラウザを閉じて再度開いてもログイン状態が維持される（7日間）
- [ ] アクセストークン期限切れ時、自動的にリフレッシュされる

#### エラーハンドリング

| エラーケース                 | HTTPステータス | メッセージ                                       | 挙動                         |
| ---------------------------- | -------------- | ------------------------------------------------ | ---------------------------- |
| メールアドレス形式不正       | 400            | 有効なメールアドレスを入力してください           | フォームにエラー表示         |
| パスワード8文字未満          | 400            | パスワードは8文字以上で入力してください          | フォームにエラー表示         |
| メールアドレス重複           | 409            | このメールアドレスは既に登録されています         | フォームにエラー表示         |
| 認証情報不一致               | 401            | メールアドレスまたはパスワードが正しくありません | フォームにエラー表示         |
| アクセストークン期限切れ     | 401            | -                                                | 自動リフレッシュ実行         |
| リフレッシュトークン期限切れ | 401            | セッションの有効期限が切れました                 | ログインページへリダイレクト |

### 4.3 Phase 2: 日記CRUD機能

#### ユーザーストーリー

| ID      | ストーリー                                                                                   | 優先度 |
| ------- | -------------------------------------------------------------------------------------------- | ------ |
| ENTRY-1 | ログイン中のユーザーとして、今日の日記を作成したい。振り返りを記録するため。                 | 必須   |
| ENTRY-2 | ログイン中のユーザーとして、過去の日記一覧を見たい。振り返りを確認するため。                 | 必須   |
| ENTRY-3 | ログイン中のユーザーとして、特定の日記を編集したい。内容を修正するため。                     | 必須   |
| ENTRY-4 | ログイン中のユーザーとして、日記を削除したい。不要な記録を消すため。                         | 必須   |
| ENTRY-5 | ログイン中のユーザーとして、カレンダーで月別の記録を見たい。どの日に記録したか把握するため。 | 必須   |

#### 機能仕様

| 機能             | 仕様                                                                                                  |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| 日記作成         | content（本文、最大10,000文字）、shortMemo（一言メモ、最大200文字）、rating（1-5）、entryDate（日付） |
| 入力要件         | content、shortMemo、ratingのいずれか1つ以上が必須                                                     |
| 一覧表示         | 新しい順、20件/ページ、ページネーション対応                                                           |
| 詳細表示         | 自分の日記のみ閲覧可能                                                                                |
| 編集             | 自分の日記のみ編集可能                                                                                |
| 削除             | 自分の日記のみ削除可能、確認ダイアログあり                                                            |
| カレンダービュー | 月単位表示、記録がある日にマーカー表示                                                                |

#### 受け入れ条件

**ENTRY-1: 日記作成**

- [ ] 本文、一言メモ、評価のいずれかを入力して保存できる
- [ ] 日付を指定できる（デフォルトは今日）
- [ ] 同じ日付に複数の日記を作成できる
- [ ] 保存後、一覧ページに遷移する

**ENTRY-2: 日記一覧**

- [ ] 自分の日記のみ表示される
- [ ] 新しい順に表示される
- [ ] 各日記のプレビュー（日付、一言メモ、評価）が表示される
- [ ] 20件を超える場合、ページネーションが表示される

**ENTRY-3: 日記編集**

- [ ] 既存の日記を編集できる
- [ ] 他のユーザーの日記は編集できない（404表示）

**ENTRY-4: 日記削除**

- [ ] 削除前に確認ダイアログが表示される
- [ ] 削除後、一覧ページにリダイレクトされる
- [ ] 他のユーザーの日記は削除できない

**ENTRY-5: カレンダービュー**

- [ ] 月単位でカレンダーが表示される
- [ ] 記録がある日にマーカーが表示される
- [ ] 日付をクリックすると、その日の日記一覧が表示される
- [ ] 前月・翌月に移動できる

#### エラーハンドリング

| エラーケース             | HTTPステータス | メッセージ                                       | 挙動                                             |
| ------------------------ | -------------- | ------------------------------------------------ | ------------------------------------------------ |
| 入力なしで保存           | 400            | 本文、一言メモ、評価のいずれかを入力してください | フォームにエラー表示                             |
| 本文が10,000文字超過     | 400            | 本文は10,000文字以内で入力してください           | フォームにエラー表示                             |
| 存在しない日記ID         | 404            | 日記が見つかりません                             | エラーページ表示                                 |
| 他ユーザーの日記アクセス | 404            | 日記が見つかりません                             | エラーページ表示（403ではなく404で情報漏洩防止） |
| 不正な日付形式           | 400            | 日付はYYYY-MM-DD形式で入力してください           | フォームにエラー表示                             |

## 8. API エンドポイント

### 8.1 認証 (`/api/auth`)

| Method | Path        | 説明           | 認証   | リクエスト                        | レスポンス                     |
| ------ | ----------- | -------------- | ------ | --------------------------------- | ------------------------------ |
| POST   | `/register` | ユーザー登録   | 不要   | `{email, password, displayName?}` | `{user, accessToken}` + Cookie |
| POST   | `/login`    | ログイン       | 不要   | `{email, password}`               | `{user, accessToken}` + Cookie |
| POST   | `/refresh`  | トークン更新   | Cookie | -                                 | `{accessToken}`                |
| POST   | `/logout`   | ログアウト     | 必要   | -                                 | `{message}`                    |
| GET    | `/me`       | 現在のユーザー | 必要   | -                                 | `{user}`                       |

### 8.2 日記 (`/api/entries`)

| Method | Path        | 説明         | 認証 | リクエスト                                    | レスポンス                           |
| ------ | ----------- | ------------ | ---- | --------------------------------------------- | ------------------------------------ |
| GET    | `/`         | 一覧取得     | 必要 | `?page=1&limit=20`                            | `{entries, pagination}`              |
| POST   | `/`         | 作成         | 必要 | `{content?, shortMemo?, rating?, entryDate}`  | `{entry}`                            |
| GET    | `/:id`      | 詳細取得     | 必要 | -                                             | `{entry}`                            |
| PUT    | `/:id`      | 更新         | 必要 | `{content?, shortMemo?, rating?, entryDate?}` | `{entry}`                            |
| DELETE | `/:id`      | 削除         | 必要 | -                                             | `{message}`                          |
| GET    | `/calendar` | カレンダー用 | 必要 | `?year=2026&month=2`                          | `{entries: [{date, count, rating}]}` |

## 10. 用語集

| 用語                   | 説明                                 |
| ---------------------- | ------------------------------------ |
| エントリ（Entry）      | 1件の日記記録                        |
